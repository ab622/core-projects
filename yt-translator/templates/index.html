API Documentation
POST /api/translate

Translate text from any language to English

Request Body:

{
  "text": "Your text here"
}

Response:

{
  "success": true,
  "original_text": "Original text",
  "translated_text": "Translated text",
  "source_language": "ar",
  "target_language": "en",
  "confidence": 0.95
}
POST /api/detect

Detect the language of the provided text

Request Body:

{
  "text": "Your text here"
}

Response:

{
  "success": true,
  "text": "Original text",
  "language_code": "ar",
  "confidence": 0.95
}
GET /api/health

Health check endpoint with performance data

Response:

{
  "success": true,
  "status": "healthy",
  "message": "Translation API is running",
  "proxy_configured": false,
  "health_check_time_ms": 12.5
}
GET /api/status

API configuration and features

Response:

{
  "success": true,
  "api_version": "1.1",
  "features": {
    "proxy_support": true,
    "performance_monitoring": true
  }
}
Performance Features
  • Processing time monitoring
  • Proxy fallback support
  • Caching optimization
  • Real-time performance metrics
Error Response Format

All error responses include timing:

{
  "success": false,
  "error": "Error type",
  "message": "Detailed error message",
  "processing_time_ms": 156.2
}
Proxy Configuration (Optional)

To use a proxy for translation requests, set the environment variable:

TRANSLATION_PROXY_URL=http://your-proxy-server:port

When configured, the API will use the proxy for ALL translation requests to ensure consistent routing through your network infrastructure.
